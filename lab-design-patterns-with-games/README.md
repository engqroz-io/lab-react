# Design Patterns with Games

Uma coleção de implementações de padrões de design em C#, cada uma baseada em temas de jogos populares para facilitar o aprendizado.

> A collection of design pattern implementations in C#, each based on popular game themes to facilitate learning.

## 🛠️ Stacks Utilizadas

> ## 🛠️ Used Stacks

Este projeto é todo em **C#** com **.NET**, focando nos padrões de design criacionais. Usei o Visual Studio para desenvolver, e cada padrão é implementado em seu próprio projeto separado. Foi uma aventura explorar como os jogos podem ensinar conceitos de programação! 🎮

> This project is all in **C#** with **.NET**, focusing on creational design patterns. I used Visual Studio to develop, and each pattern is implemented in its own separate project. It was an adventure to explore how games can teach programming concepts! 🎮

### Padrões Implementados

| Padrão               | Tema          | Descrição                                                                                    | Link                                       |
| -------------------- | ------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------ |
| **Abstract Factory** | StarCraft     | Criação de famílias de objetos relacionados (bases, revestimentos, sistemas de energia)      | [Ver Detalhes](abstract-factory/README.md) |
| **Factory Method**   | Mortal Kombat | Interface para criação de objetos sem especificar classes concretas (seleção de personagens) | [Ver Detalhes](factory-method/README.md)   |
| **Builder**          | Army Builder  | Construção passo a passo de objetos complexos (criação de exércitos)                         | `builder/`                                 |
| **Prototype**        | Cloud Manager | Clonagem de objetos para evitar recriação (gerenciamento de nuvens)                          | `prototype/`                               |
| **Singleton**        | Game Manager  | Garantia de uma única instância de um objeto em toda a aplicação                             | `singleton/`                               |
